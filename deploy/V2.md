# Deploying Duck Pond v2 as a SEPARATE service

v2 is a fresh, fully separate Cloud Run + Cloud SQL stack. The live v1 deploy is
**never touched**. The whole separation is driven by one env var: `SERVICE_PREFIX`.

`deploy/deploy.sh` namespaces every resource by `SERVICE_PREFIX` (default
`duckpond` = v1). Set it to `duckpond-v2` and v2 deploys alongside v1:

| Resource                 | v1 (default)        | v2 (`SERVICE_PREFIX=duckpond-v2`) |
|--------------------------|---------------------|-----------------------------------|
| Cloud Run API service    | `duckpond-api`      | `duckpond-v2-api`                 |
| Cloud Run web service    | `duckpond-web`      | `duckpond-v2-web`                 |
| Cloud SQL instance       | `duckpond-db`       | `duckpond-v2-db`                  |
| Secret: DATABASE_URL     | `duckpond-db-url`   | `duckpond-v2-db-url`              |
| Secret: JWT_SECRET       | `duckpond-jwt`      | `duckpond-v2-jwt`                 |
| Secret: ANTHROPIC_API_KEY| `duckpond-anthropic`| `duckpond-v2-anthropic`           |

The Artifact Registry repo (`duckpond`), region (`europe-west2`), GCP project and
Google OAuth client (hosted-domain lock) are shared — same as v1.

## One-time prerequisites for v2

Before the first v2 deploy, create the v2 Cloud SQL instance and the three v2
secrets (names from the table above). Example:

```bash
# Cloud SQL (Postgres) for v2 — separate instance, does not touch v1
gcloud sql instances create duckpond-v2-db \
  --database-version=POSTGRES_15 --edition=ENTERPRISE \
  --tier=db-f1-micro --region=europe-west2 --project "$PROJECT_ID"
gcloud sql databases create duckpond --instance duckpond-v2-db --project "$PROJECT_ID"
# ...create the duck user + password, then store the connection string:
printf '%s' "postgresql+psycopg://duck:PASSWORD@/duckpond?host=/cloudsql/${PROJECT_ID}:europe-west2:duckpond-v2-db" \
  | gcloud secrets create duckpond-v2-db-url --data-file=- --project "$PROJECT_ID"
printf '%s' "$(openssl rand -hex 32)" | gcloud secrets create duckpond-v2-jwt       --data-file=- --project "$PROJECT_ID"
printf '%s' "sk-ant-..."             | gcloud secrets create duckpond-v2-anthropic --data-file=- --project "$PROJECT_ID"
```

## Deploy v2 (EXACT steps)

```bash
export SERVICE_PREFIX=duckpond-v2
export PROJECT_ID=<gcp-project-id>
export HOSTED_DOMAIN=bikeluggage.co.uk
export ADMIN_EMAILS=callum@bikeluggage.co.uk
export GOOGLE_CLIENT_ID=<oauth-client-id>
# REGION defaults to europe-west2; SQL_INSTANCE/secrets derive from SERVICE_PREFIX

bash deploy/deploy.sh
```

The script prints the v2 web + API URLs at the end.
